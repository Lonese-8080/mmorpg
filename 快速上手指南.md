## 📌 这是什么？

这是一个 MMORPG（大型多人在线角色扮演游戏）服务器框架。简单说就是：
- ✅ 可以同时支持很多玩家在线
- ✅ 运行稳定，不容易崩溃
- ✅ 有监控系统，可以看到运行状态
- ✅ 有自动测试，保证代码质量

## 🚀 最快的开始方式（3步）

### 第1步：启动服务

在服务器上执行：

```bash
cd ~/mmorpg
bash deploy.sh
```

然后会出现一个菜单，输入 `1` 然后按回车，服务就启动了！

### 第2步：查看测试结果

打开浏览器访问：
```
http://你的服务器IP:8080/test/results
```

可以看到所有测试的通过/失败状态。

### 第3步：查看监控（Grafana）

打开浏览器访问：
```
http://你的服务器IP:3000
```

账号：`admin`  
密码：`admin123`（生产环境请修改）

### 二、本地开发机怎么用？（Windows）

#### 1. 编译项目
```powershell
cd G:\QZWCS\MMORPG
dotnet build MMORPG.sln -c Release
```

#### 2. 运行测试
```powershell
dotnet test tests/MMORPG.Framework.Tests/MMORPG.Framework.Tests.csproj
```

#### 3. 构建 Docker 镜像
```powershell
docker compose build --no-cache framework-test
```

#### 4. 使用一键部署脚本
```powershell
.\deploy.ps1 -Action build      # 只编译
.\deploy.ps1 -Action test       # 只测试
.\deploy.ps1 -Action docker     # 只构建镜像
```

---

### 三、更新代码后怎么部署到服务器？

#### 方法一：上传单个文件（推荐，最快）

如果只改了一两个文件：

```powershell
# 本地 PowerShell
scp 修改的文件路径 lonese@lonese-mmorpg:~/mmorpg/对应路径
```

然后到服务器上执行：
```bash
cd ~/mmorpg
bash deploy.sh
# 选择 3（重新构建并启动）
```

#### 方法二：完整上传

如果改了很多文件，用 Git 最方便：

```powershell
# 本地提交代码
git add .
git commit -m "修改了什么什么"
git push
```

然后到服务器上拉取：
```bash
cd ~/mmorpg
git pull
bash deploy.sh
# 选择 3
```

## 🔍 常见问题

### Q1: 服务启动不起来怎么办？

**步骤1**：查看日志
```bash
cd ~/mmorpg
bash deploy.sh
# 选择 5，再选择 1
```

**步骤2**：看看有没有明显的错误信息
- 端口被占用？换个端口
- 内存不够？关掉其他程序
- 配置文件错了？检查 config 目录

### Q2: 测试失败了怎么办？

**步骤1**：看测试结果页面
```
http://服务器IP:8080/test/results
```

**步骤2**：找到失败的测试，看错误信息

**步骤3**：如果是代码问题，修复后重新部署

### Q3: 服务器卡怎么办？

**步骤1**：查看资源使用
```bash
top
```
按 `q` 退出

**步骤2**：如果 CPU 或内存很高
- 减少连接数限制
- 降低帧率
- 增加服务器配置

### Q4: 怎么修改端口？

编辑 `docker-compose.yml`，找到 ports 部分：
```yaml
ports:
  - "7001:7001"   # 左边是宿主机端口，右边是容器端口
  - "8080:8080"
```

把左边的数字改成你想要的端口，然后重启服务。

### Q5: 忘记 Grafana 密码了？

执行以下命令重置：
```bash
cd ~/mmorpg
docker exec -it mmorpg-grafana grafana-cli admin reset-admin-password 新密码
```

## 💡 小技巧

1. **用部署脚本** - `deploy.sh` / `deploy.ps1` 包含了常用操作，不用记命令
2. **先备份** - 改配置前先备份，出问题可以快速回滚
3. **看日志** - 出问题先看日志，90% 的问题日志里都有答案
4. **小步迭代** - 一次只改一个东西，改完测一下
