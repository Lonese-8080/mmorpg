## 1. 概述

### 1.1 目的

```
本规范旨在：
1. 提高代码可读性和可维护性
2. 减少 bug 的发生
3. 提高开发效率
4. 便于团队协作
```

### 1.2 适用范围

```
适用于：
- MMORPG 框架的所有源代码
- 包括 Framework、Core、Game 层

不适用于：
- 第三方库代码
- 自动生成的代码
```

## 3. 代码格式

### 3.1 缩进和空格

```csharp
// ============================================================
// ✅ 正确示例
// ============================================================

// 大括号单独一行
public void Method()
{
    if (condition)
    {
        DoSomething();
    }
}

// 运算符周围加空格
var result = a + b * c;

// 关键字和括号之间加空格
if (condition)
{
}

// 逗号后面加空格
Method(1, 2, 3);

// 数组索引不加空格
array[0] = value;

// 方法调用不加空格
Method(args);

// ============================================================
// ❌ 错误示例
// ============================================================

// 大括号不单独一行
public void Method(){
    if (condition){
        DoSomething();
    }
}

// 缺少空格
var result = a+b*c;

// 关键字和括号之间缺空格
if(condition){
}

// 逗号后面缺空格
Method(1,2,3);
```

### 3.2 空行

```csharp
// ============================================================
// ✅ 正确示例
// ============================================================

// 使用空行分隔逻辑段落
public void Method1()
{
    var a = 0;
    a = CalculateA();
    
    var b = 0;
    b = CalculateB();
}

// 类成员之间用空行分隔
public class Example
{
    // 字段
    private int _field;
    
    // 属性
    public int Property { get; set; }
    
    // 构造函数
    public Example()
    {
    }
    
    // 方法
    public void Method()
    {
    }
}

// 命名空间引用之间用空行
using System;
using System.Collections.Generic;

using MMORPG.Framework;
using MMORPG.Core;
```

### 3.3 行长度

```csharp
// ============================================================
// ✅ 正确示例
// ============================================================

// 每行不超过 120 个字符
// 如果太长，使用以下方式拆分

// 1. 方法参数换行
public void Method(
    string parameter1,
    int parameter2,
    long parameter3)
{
}

// 2. 链式调用换行
var result = someObject
    .Method1()
    .Method2()
    .Method3();

// 3. 复杂条件换行
if (condition1 &&
    condition2 &&
    condition3)
{
}

// 4. 字符串拼接
var message = "这是一段很长的消息，"
    + "用于说明如何处理长字符串，"
    + "以及何时应该换行。";
```

    }

## 5. 最佳实践

### 5.1 类型设计

```csharp
// ============================================================
// ✅ 正确示例
// ============================================================

// 1. 使用 readonly 字段
private readonly int _maxConnections;

// 2. 使用 const 代替 magic number
private const int MaxBufferSize = 1024;

// 3. 使用 string.Empty 代替 ""
var name = string.Empty;

// 4. 使用可空类型明确表示可空
public int? NullableId { get; set; }

// 5. 使用 var 时类型应该显而易见
var message = "Hello";  // string 显而易见
var list = new List<int>();  // List 显而易见
var count = GetCount();  // count 显而易见

// 但下面的情况不用 var：
int value = 100;  // 明确类型
Stream stream = CreateStream();  // 基类更合适
IMessage message = new LoginMessage();  // 接口更合适

// ============================================================
// ❌ 错误示例
// ============================================================

// 1. 不要用 object
var data = new object();  // ❌ 不知道能做什么

// 2. 不要返回 null（尽量）
public List<Item> GetItems()
{
    return null;  // ❌ 改用空列表
}

// 3. 不要用 ArrayList（用泛型）
var list = new ArrayList();  // ❌ 用 List<T>
```

### 5.2 方法设计

```csharp
// ============================================================
// ✅ 正确示例
// ============================================================

// 1. 方法不要太长（不超过 50 行）
// 如果太长，拆分成多个方法

// 2. 参数不要太多（不超过 5 个）
// 如果太多，使用参数对象
public void CreatePlayer(string name, int level, int profession, 
    string account, long playerId)
{
    // ❌ 参数太多，拆分成参数对象
}

public void CreatePlayer(CreatePlayerRequest request)
{
    // ✅ 使用参数对象
}

// 3. 使用 async/await 代替回调
public async Task<string> GetDataAsync()
{
    var result = await HttpClient.GetStringAsync(url);
    return result;
}

// 4. 方法职责单一
// 一个方法只做一件事
public void ValidateAndSave()
{
    // ❌ 这个方法做了两件事
}

public void Validate()
{
    // ✅ 验证
}

public void Save()
{
    // ✅ 保存
}

// ============================================================
// ❌ 错误示例
// ============================================================

// 1. 不要有 out 参数（除非必要）
public bool TryParse(string input, out int result)
{
    // 这个是可以接受的（Try 模式）
}

// 但下面的情况不要用 out：
public void Method(string input, out int a, out int b, out int c)
{
    // ❌ out 参数太多
}

// 2. 不要用 ref（除非必要）
public void Method(ref int value)
{
    // ❌ ref 使用要谨慎
}
```

### 5.3 异常处理

```csharp
// ============================================================
// ✅ 正确示例
// ============================================================

// 1. 使用具体的异常类型
try
{
    await SaveToDatabaseAsync();
}
catch (DbConnectionException ex)
{
    Logger.Error("数据库连接失败", ex);
    throw;  // 或者转换异常
}
catch (Exception ex)
{
    Logger.Error("保存失败", ex);
    throw new SaveFailedException("无法保存数据", ex);
}

// 2. 不要吞掉异常
try
{
    DoSomething();
}
catch (Exception)
{
    // ❌ 永远不要这样做
}

// 应该至少记录日志
catch (Exception ex)
{
    Logger.Error(ex);
}

// 3. 使用 using 确保资源释放
using var stream = new FileStream(path, FileMode.Open);
var data = ReadFromStream(stream);
// stream 会自动关闭

// 4. 不要在 finally 中抛出异常
finally
{
    // ❌ 如果这里抛异常，会覆盖原始异常
    throw new CleanupException();
}
```

## 6. 性能规范

### 6.1 内存分配

```csharp
// ============================================================
// ✅ 正确示例
// ============================================================

// 1. 使用对象池
var buffer = ArrayPool<byte>.Shared.Rent(1024);
try
{
    // 使用 buffer
}
finally
{
    ArrayPool<byte>.Shared.Return(buffer);
}

// 2. 使用 Span<T> 避免分配
public unsafe string GetString(Span<byte> data)
{
    // 不需要额外的 byte[] 分配
}

// 3. 使用 StringBuilder 拼接长字符串
var sb = new StringBuilder();
sb.Append("第一部分");
sb.Append("第二部分");
var result = sb.ToString();

// 4. 使用 List<T> 预分配容量
var list = new List<int>(1000);  // 预分配 1000 个元素的空间

// ============================================================
// ❌ 错误示例
// ============================================================

// 1. 字符串拼接（在循环中）
var result = "";
for (int i = 0; i < 1000; i++)
{
    result += i.ToString();  // ❌ 每次都分配新字符串
}

// 2. 在循环中创建对象
for (int i = 0; i < 1000; i++)
{
    var message = new Message();  // ❌ 1000 次分配
}

// 3. 使用不必要的 LINQ
var result = list.Where(x => x > 0).Select(x => x * 2).ToList();
// ❌ 在性能敏感代码中不要用复杂 LINQ

// 4. 不要在循环中访问集合的 Count
for (int i = 0; i < list.Count; i++)  // ❌ 每次迭代都访问 Count
{
}
var count = list.Count;  // ✅ 先缓存
for (int i = 0; i < count; i++)
{
}
```

### 6.2 循环优化

```csharp
// ============================================================
// ✅ 正确示例
// ============================================================

// 1. 使用 for 循环代替 foreach（对于数组）
var array = new int[1000];
for (int i = 0; i < array.Length; i++)
{
    // ✅ for 比 foreach 快
}

// 2. 逆序循环（有时更快）
for (int i = array.Length - 1; i >= 0; i--)
{
}

// 3. 循环展开（对于简单操作）
for (int i = 0; i < length; i += 4)
{
    Process(array[i]);
    Process(array[i + 1]);
    Process(array[i + 2]);
    Process(array[i + 3]);
}

// 4. 提前退出
foreach (var item in items)
{
    if (item.IsTarget)
    {
        result = item;  // ✅ 找到就退出
        break;
    }
}

// ============================================================
// ❌ 错误示例
// ============================================================

// 嵌套过深
for (int i = 0; i < 10; i++)
{
    for (int j = 0; j < 10; j++)
    {
        for (int k = 0; k < 10; k++)
        {
            // ❌ 嵌套太深，难以维护
        }
    }
}
```

## 7. 安全性规范

### 7.1 输入验证

```csharp
// ============================================================
// ✅ 正确示例
// ============================================================

// 1. 验证所有外部输入
public void ProcessMessage(string message)
{
    if (string.IsNullOrEmpty(message))
    {
        throw new ArgumentNullException(nameof(message));
    }
    
    if (message.Length > MaxMessageLength)
    {
        throw new ArgumentException("消息过长");
    }
    
    // 处理消息
}

// 2. 使用参数化查询（防 SQL 注入）
using var command = connection.CreateCommand();
command.CommandText = "SELECT * FROM Users WHERE Account = @account";
command.Parameters.AddWithValue("@account", account);

// 3. 验证数值范围
if (level < 1 || level > MaxLevel)
{
    throw new ArgumentOutOfRangeException(nameof(level));
}

// ============================================================
// ❌ 错误示例
// ============================================================

// 1. 直接拼接 SQL
var sql = "SELECT * FROM Users WHERE Account = '" + account + "'";
// ❌ SQL 注入漏洞！

// 2. 不验证外部输入
public void SetHealth(int health)
{
    _health = health;  // ❌ health 可能是负数或超大值
}

// 3. 信任配置文件
var apiKey = Configuration["ApiKey"];  // ❌ 配置也可能是恶意的
```
