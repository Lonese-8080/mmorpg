# Performance Tuning Guide

This guide covers performance optimization techniques for MMORPG Framework in production environments.

---

## 1. Network Layer Optimization

### 1.1 TCP Buffer Sizes

```yaml
# appsettings.json
{
  "Network": {
    "ReceiveBufferSize": 65536,    # 64KB receive buffer
    "SendBufferSize": 65536,      # 64KB send buffer
    "MaxPendingConnections": 1000  # Max pending connection queue
  }
}
```

**Recommendations:**
- Increase buffer sizes for high-throughput servers
- Monitor buffer overflow metrics
- Use `ss -s` to check socket memory usage

### 1.2 Connection Pooling

```yaml
{
  "Network": {
    "EnableSessionPool": true,
    "MaxPooledSessions": 100,
    "SessionPoolMinSize": 10
  }
}
```

### 1.3 Message Queue Tuning

```yaml
{
  "Network": {
    "MessageQueueCapacity": 10000,
    "MessageRetryCount": 3,
    "MessageRetryDelayMs": 100
  }
}
```

## 2. Game Scheduler Optimization

### 2.1 Target FPS

```yaml
{
  "GameScheduler": {
    "TargetFps": 30,           # Target frames per second
    "MaxFrameTimeMs": 100,     # Max time per frame (for catchup)
    "EnableAdaptiveFps": true   # Adaptive frame rate
  }
}
```

**FPS Recommendations:**
| Server Type | Target FPS | Use Case |
|------------|-----------|----------|
| Combat | 60 | Fast-paced combat |
| Social | 30 | MMO chat, social features |
| BattleRoyale | 60 | Real-time PvP |
| TurnBased | 10 | Turn-based games |

### 2.2 Frame Time Budget

```yaml
{
  "GameScheduler": {
    "FrameTimeWarningThresholdMs": 20,
    "FrameTimeCriticalThresholdMs": 50
  }
}
```

## 3. Memory Optimization

### 3.1 Object Pooling

```yaml
{
  "Memory": {
    "EnableObjectPool": true,
    "MaxPooledObjects": 10000,
    "PooledObjectTypes": [
      "Network.Message",
      "Game.Player",
      "Combat.Ability"
    ]
  }
}
```

### 3.2 GC Tuning

```bash
# Set GC mode
export DOTNET_gcServer=1
export DOTNET_gcConcurrent=1

# Heap size limits
export DOTNET_Heap_DHardLimit=8589934592  # 8GB hard limit
export DOTNET_Heap_TargetSize=6442450944  # 6GB target
```

### 3.3 Memory Profiling

```csharp
// Enable diagnostics
services.AddMemoryDiagnostics();

// Monitor memory
var memoryStats = GC.GetGCMemoryInfo();
Console.WriteLine($"Heap: {memoryStats.HeapSizeBytes / 1024 / 1024} MB");
```

# Enable SIMD in .NET

```bash
# Pin process to specific cores
taskset -c 0-7 dotnet mmorpg.dll

```

## 6. Monitoring & Profiling

### 6.1 Enable Performance Counters

```yaml
{
  "Observability": {
    "EnablePerformanceCounters": true,
    "CounterUpdateIntervalMs": 1000
  }
}
```

### 6.2 CPU Profiling

```bash
# Use dotnet-counters
dotnet-counters monitor -n MMORPG.Framework

# Use dotnet-trace
dotnet-trace collect -p <pid> --providers Microsoft-DotNETCore-EventCounter
```

### 6.3 Memory Profiling

```bash
# Use dotnet-gcdump
dotnet-gcdump collect -p <pid>

# Analyze with dotnet-dump
dotnet-dump analyze <dump-file>
```

## 7. Linux-Specific Optimizations

### 7.1 Kernel Parameters

```bash
# /etc/sysctl.conf

# Network
net.core.somaxconn=1024
net.core.netdev_max_backlog=5000
net.ipv4.tcp_max_syn_backlog=2048

# Memory
vm.swappiness=10
vm.max_map_count=262144

# File descriptors
fs.file-max=65536
```

Apply changes:
```bash
sudo sysctl -p
```

### 7.2 ulimit Settings

```bash
# /etc/security/limits.conf
* soft nofile 65536
* hard nofile 65536
* soft nproc 32768
* hard nproc 32768
```

### 7.3 Transparent Huge Pages

```bash
# Disable for better performance
echo never > /sys/kernel/mm/transparent_hugepage/enabled
echo never > /sys/kernel/mm/transparent_hugepage/defrag
```

## 9. Troubleshooting

### High CPU Usage

1. Check frame time distribution
2. Profile hot paths with dotnet-trace
3. Review GC statistics

### High Memory Usage

1. Enable memory diagnostics
2. Check for memory leaks with dotnet-gcdump
3. Review object pool utilization

### High Latency

1. Check network buffer utilization
2. Review message queue depth
3. Profile database queries
