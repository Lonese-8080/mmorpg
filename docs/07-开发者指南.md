## ⚠️ 实现状态说明

本文档描述的开发指南基于 **现有框架代码中已实现的模块**：

- `MMORPG.Framework.Network` — 网络层（TcpServer、Session、SessionPool、MessageRetryQueue、MessageRouter、MessageSerializer）
- `MMORPG.Framework.Threading` — 线程调度、消息队列、雪花算法
- `MMORPG.Framework.Logging` — 结构化日志
- `MMORPG.Framework.Configuration` — 配置加载、加密、热更新
- `MMORPG.Framework.Resilience` — 弹性策略（超时、重试、熔断）
- `MMORPG.Framework.Observability` — 指标收集、健康检查、崩溃报告、内存监控、Prometheus 导出
- `MMORPG.Framework.Security` — 消息限流（RateLimiter）、服务状态管理（ServiceStateManager）

**本文档中以下章节描述的内容为 **规划**（尚未实现）**：
- `MMORPG.Core.ECS` — World、Entity、Component、System
- `MMORPG.Framework.Serialization` — Protobuf 编解码器（**已实现**，由 `Google.Protobuf` 提供）
- 数据持久化层、场景系统等

** 注意：示例代码中出现的 `World`、`Entity`、`Component`、`System`、`ProtobufHelper`、`MMORPG.Framework.Serialization` 等 API **尚未实现 **。请参考对应设计文档了解设计思路。**

### 1.1 安装 .NET 10 SDK

```
步骤：

1. 下载 .NET 10 SDK
   地址：https://dotnet.microsoft.com/download/dotnet/10.0

2. 安装

   - Linux：参考官方文档

3. 验证安装
```

```
步骤：

1. 下载 Visual Studio 2022
   地址：https://visualstudio.microsoft.com/downloads/

2. 安装时选择以下工作负载：
   - .NET 桌面开发
   - 使用 C++ 的桌面开发（如果需要 native interop）

3. 验证
   打开 VS，创建新项目，搜索 "Console App"
   确保有 .NET 10 选项
```

```bash

# 还原依赖
```

## 4. 模块开发指南

### 4.1 Framework.Resilience（弹性策略）

**场景**：为数据库访问添加自动重试、熔断和超时保护。

```csharp
// 1. 创建弹性管道（3 行代码）
var pipeline = new ResiliencePipelineBuilder()
    .WithTimeout("DBTimeout", new TimeoutOptions { TimeoutMs = 3000 })
    .WithRetry("DBRetry", new RetryOptions
    {
        MaxRetryCount = 3,
        BaseDelayMs = 200,
        ExponentialBase = 2,
        UseJitter = true
    })
    .WithCircuitBreaker("DBCircuit", new CircuitBreakerOptions
    {
        FailureThreshold = 5,
        DurationOfBreakMs = 30_000
    })
    .Build();

// 2. 为消息处理器附加管道
MessageRouter.Instance.RegisterHandler(
    (uint)MessageIds.C2S_PlayerLogin,
    async (session, msg) => {
        // 此调用受管道保护：自动重试、熔断、超时
        var result = await pipeline.ExecuteAsync(async ct =>
            await _playerService.LoginAsync((PlayerLoginRequest)msg, ct));
        return new PlayerLoginResponse { Success = true, Token = result.Token };
    },
    pipeline: pipeline);  // ← 注入弹性策略
```

### 4.2 Framework.Observability（可观测性）

**场景**：启动指标收集、健康检查端点和崩溃报告。

```csharp
// 1. 框架统一初始化（推荐方式）
FrameworkBootstrap.Configure(FrameworkOptions.CreateProduction());

// 或者逐个启用：
MetricsCollector.Instance.Enable();
HealthCheckService.Instance.AddMemoryCheck(1024L * 1024 * 1024); // 1GB
CrashReporting.Enable("logs/crashes");

// 2. 注册业务指标
var loginCounter = MetricsCollector.Instance.RegisterCounter(
    "login_total", "累计登录次数");
loginCounter?.Increment();

// 3. 启动健康检查 HTTP 端点
var healthEndpoint = new HealthEndpoint(port: 8080);
healthEndpoint.Start();

// 4. 启动 Prometheus 导出器
var prometheus = new PrometheusExporter(port: 9091);
prometheus.Start();
```

### 4.3 Framework.Security（安全）

**场景**：为单玩家设置消息频率限制，或全局设置服务器限流。

```csharp
// 1. 全局限流（整服每秒 10000 条消息）
MessageRouter.Instance.SetGlobalRateLimit(10000);

// 2. 会话级限流（单玩家每秒 100 条）
session.SetSessionRateLimit(100);

// 3. 独立 RateLimiter（更细粒度控制）
var attackLimiter = new RateLimiter(5); // 每秒最多 5 次攻击
if (attackLimiter.TryAcquire())
{
    ProcessAttack(session, (AttackRequest)msg);
}
// else: 丢弃请求

// 4. 服务状态管理（优雅暂停）
await ServiceStateManager.Instance.PauseAsync();
// 此时 MessageRouter 只处理心跳和内部消息

await ServiceStateManager.Instance.ResumeAsync();
```

### 4.4 Framework.Configuration（配置热更新）

**场景**：从 JSON 文件加载配置，支持敏感字段自动解密和热更新。

```csharp
// 1. 加载配置（自动解密 ENC[v1]: 开头的敏感字段）
var config = ConfigurationLoader.Load<ServerConfig>(
    "config/appsettings.json",
    enableEnvironmentVariables: true);

// 2. 启用配置热更新（文件变更时自动重载）
ConfigurationLoader.EnableFileWatcher<ServerConfig>(
    "config/appsettings.json",
    intervalMs: 1000,
    envVars: true);

// 3. 订阅配置变更通知
ConfigurationLoader.ConfigChanged += (sender, args) =>
{
    Logger.Info("Config", "配置已更新: {0}", args.FilePath);
    // 重新应用配置到服务
};

// 4. 手动重载
ConfigurationLoader.Reload<ServerConfig>("config/appsettings.json", true);

// 5. 使用 ConfigCenter（配置中心）
ConfigCenter.Initialize("config/server.json");
ConfigCenter.OnChange<ServerConfig>((oldCfg, newCfg) =>
{
    Logger.Info("Config", "Network.Port: {0} → {1}",
        oldCfg?.Network.Port, newCfg.Network.Port);
});
```

### 4.3 添加新的消息处理器

```csharp
// ⚠️ 说明：当前框架使用 Google Protobuf 序列化。
//        消息类由 .proto 文件在编译时自动生成。
//
// 步骤1：在 protos/ 目录下定义 .proto 消息

// protos/game.proto:
//
// syntax = "proto3";
// package mmorpg.game;
//
// message C2S_Xxx {
//     int32  int_field  = 1;
//     string str_field  = 2;
// }

// 编译后自动生成到 Network/Protobuf/Game.cs：
//
// using pb = Google.Protobuf;
//
// public sealed partial class C2S_Xxx : pb::IMessage<C2S_Xxx>
// {
//     public int IntField { get; set; }
//     public string StrField { get; set; }
//     // ...
// }

// 步骤1b：在 MessageId.cs 中分配消息 ID
//
// public static partial class MessageIds
// {
//     public const uint C2S_Xxx = 0x00001000;  // 业务层从 0x1000 开始
// }

// 步骤1c：在 MessageSerializer 中注册（框架启动时自动注册）
//
// MessageSerializer.Register<C2S_Xxx>(MessageIds.C2S_Xxx);

// 步骤2：创建消息处理器

/// <summary>
/// 消息处理器 —— 实现 IMessageHandler<C2S_Xxx> 接口
/// </summary>
public class XxxHandler : IMessageHandler<C2S_Xxx>
{
    /// <summary>
    /// 处理消息
    /// </summary>
    public async Task HandleAsync(Session session, C2S_Xxx message)
    {
        // 验证参数
        if (string.IsNullOrEmpty(message.StrField))
        {
            await session.SendAsync(new S2C_Error  // S2C_Error 为框架内置错误响应消息
            {
                ErrorCode = 1,
                ErrorMessage = "参数错误"
            });
            return;
        }

        // 业务逻辑
        var result = await ProcessAsync(message);

        // 发送响应
        await session.SendAsync(result);
    }

    private async Task<S2C_XxxResult> ProcessAsync(C2S_Xxx message)
    {
        // 业务处理
        await Task.CompletedTask;
        return new S2C_XxxResult { Success = true };
    }
}

// 步骤3：注册消息类到序列化器（程序启动时）
MessageSerializer.Register<C2S_Xxx>(0x00001000);
MessageSerializer.Register<S2C_XxxResult>(0x00001001);

// 步骤4：注册处理器到 MessageRouter（单例模式）
MessageRouter.Instance.RegisterHandler(new XxxHandler());

// 步骤5：在 TcpServer 的 OnMessageReceived 事件中调用 Route
server.OnMessageReceived += (session, message) =>
{
    MessageRouter.Instance.Route(session, message);
};
```

## 6. 常见问题

### Q1: 编译报错 "找不到 xxx"

```
可能原因：
1. 没有还原 NuGet 包
   解决：dotnet restore

2. 项目引用错误
   解决：检查 .csproj 中的 <ProjectReference>

3. 生成顺序问题
   解决：dotnet build MMORPG.sln
```

### Q2: 运行时提示 "找不到消息类型"

```
可能原因：
- 自定义消息类没有通过 MessageSerializer.Register<T>() 注册

解决：
- 确保消息类在 protos/*.proto 中定义，并已注册 MessageSerializer.Register<T>(messageId)
- 在程序启动时调用 MessageSerializer.Register<T>(messageId)
- 确认消息 ID 不与框架消息 ID 冲突（框架 ID 范围：0x00000001 ~ 0x00000FFF）
```

### Q3: 消息处理器没有被调用

```
可能原因：
1. 没有注册处理器
   解决：MessageRouter.Instance.RegisterHandler(new XxxHandler());

2. 消息类型不匹配
   解决：检查 C# 消息类的 MessageId 是否与注册时一致
         确保 IMessageHandler<T> 的泛型参数与消息类匹配

3. 网络层没有触发 Route
   解决：检查 TcpServer 的 OnMessageReceived 事件是否绑定
         确保在事件回调中调用 MessageRouter.Instance.Route(session, message)
```

### Q4: 内存持续增长

```
可能原因：
1. Session 没有正确关闭
   解决：确保异常时调用 session.Disconnect(reason)

2. MessageChannel 消费者速度低于生产者
   解决：检查 DrainAll() 是否被持续调用
         检查主循环帧率是否过低

3. 日志队列堆积
   解决：确保 Logger.ShutdownAsync() 在程序退出前被调用
```
