## 版本历史

### v3.0.0 (2026-06-23)

#### Metrics 模块全面现代化 —— 迁移至 System.Diagnostics.Metrics（#10）

本次版本将指标模块从完全自研实现迁移至 .NET 官方的 `System.Diagnostics.Metrics`（Meter API），
大幅提升性能并接入 .NET 生态。这是一个**架构级优化**，对外 API 完全兼容。

**1. 架构变化**

| 组件 | 旧实现 | 新实现 |
|------|--------|--------|
| 计数器 | `Interlocked` + 自定义 | `Counter<long>`（Meter API）+ 本地跟踪 |
| 直方图 | lock + 环形缓冲 | `Histogram<double>`（Meter API）+ 环形缓冲（分位数） |
| 量规 | `Interlocked` + BitConverter | `ObservableGauge<double>`（Meter API）+ 本地跟踪 |
| 指标收集器 | `ConcurrentDictionary` 管理 | 新增 `Meter`（名称：`MMORPG.Framework`） |

**2. 主要收益**

- **性能**：Meter API 底层由 .NET 团队高度优化（ThreadLocal 分片 + 无锁聚合），高并发下性能优于手动 Interlocked
- **生态兼容**：外部工具（dotnet-counters、OpenTelemetry、Prometheus、Grafana、SkyWalking 等）
  可直接通过 Meter 名称 `MMORPG.Framework` 订阅所有指标
- **零分配 Hot Path**：Meter API 写入路径零分配
- **向后兼容**：`ICounter` / `IHistogram` / `IGauge` / `MetricsCollector` 接口完全不变，
  业务层调用无需修改
- **Histogram 分位数**：保留环形缓冲实现 P50/P95/P99（Meter Histogram 本身不暴露分位数，
  分位数聚合由外部 Listener 负责）

**3. 测试结果**

- Framework.Tests: **187/187 通过**
- Framework.IntegrationTests: **10/10 通过**
- 编译: 0 错误

### v0.3.0 (2026-06-21)

#### 线程模型 / 调度器 / 雪花算法 完成

**新增功能：**

- `MMORPG.Framework.Threading` 命名空间
  - `SnowflakeIdGenerator` - 雪花算法 ID 生成器（64位，41位时间戳+10位机器ID+12位序列号）
    - 支持自定义起始时间、机器 ID、位数配置
    - 内置时间回拨防护（自旋等待/强制时间戳）
    - 提供 `IdGenerator` 快捷工厂类
    - 支持 ID 解析（反推时间、来源节点）
  - `MessageChannel<T>` - 线程安全消息队列（基于 .NET Channel）
    - 支持无界 `CreateUnbounded<T>()` 和有界 `CreateBounded<T>(int capacity)`
    - 内部 `Interlocked` 计数器维护消息数量
    - 支持批量读取 `DrainAll()` / `DrainUpTo(int max)`
    - 多生产者单消费者模型（MPSC）
  - `GameScheduler` - 游戏主循环调度器
    - 固定帧率（20-240Hz 可配置）
    - 三阶段事件：`OnFrameStart` → `OnUpdate` → `OnFrameEnd`
    - `FrameEventArgs` 包含帧号、delta time、帧时间
    - FPS 统计（最近 100 帧滑动窗口）
    - Stopwatch 精确控制帧间隔
    - 提供 `StartAsync()` 异步启动方式
  - `ThreadSafeCollections` - 线程安全集合工具类
    - `AtomicCounter` - 原子计数器
    - `ConcurrentDictionary<TKey, TValue>` - 并发字典（隐藏基类方法）

- 单元测试（75 个，全部通过）
  - `ThreadingTests.cs` - 消息队列、调度器、集合工具（10+ 测试）
  - `SnowflakeIdGeneratorTests.cs` - 雪花算法（10+ 测试）
    - 唯一性测试（单线程/多线程）
    - 递增性测试
    - 时间戳解析
    - 并发压力测试（8 线程生成 100 万 ID，0 重复）
  - `MessageChannelTests.cs` - 消息队列（12+ 测试）
    - 写入/读取
    - 批量操作
    - 有界队列
    - 多线程并发
  - `Network/MessageRouterTests.cs` - 消息路由

**技术规格：**

| 项目 | 指标 |
|------|------|
| 雪花算法节点数 | 最大 1024 |
| 单节点 ID 生成量 | ~400 万/秒 |
| ID 总位数 | 64 位 (long) |
| 调度器帧率范围 | 20-240 Hz |
| 调度器统计窗口 | 最近 100 帧滑动平均 |
| 单元测试总数 | 75（全部通过） |
| 测试运行时间 | ~2 秒 |
